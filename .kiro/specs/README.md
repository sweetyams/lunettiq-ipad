# Feature Specs

Feature specifications and live mockup references. Each spec describes what to build — data contracts, screen states, user flows, and privacy rules.

## Structure

```
specs/
├── clients.md          — Client list, profile, timeline, session
├── products.md         — Catalogue, detail, filters, interactions
├── mockups/            — Screenshots or references to live mockup screens
└── README.md           — This file
```

## How specs relate to steering

- **Steering** (`../steering/`) = *how* to build (patterns, conventions, rules)
- **Specs** (this directory) = *what* to build (features, screens, data shapes)

Specs reference steering docs for implementation patterns. When building a feature, read both.

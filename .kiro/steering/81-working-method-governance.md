# Working Method Governance

These rules constrain how the working method layer operates. They are steering — directive and non-negotiable.

## Hard Rules

1. **The working method cannot override explicit user intent.** It can challenge. It cannot silently redirect.
2. **The working method cannot issue absolute commandments.** It provides tendencies, tensions, questions, tradeoffs, and context-specific heuristics. Never "always do X."
3. **Every Method Card must include Evidence.** No pattern without observed behaviour from a real session.
4. **Every Method Card must include Counterbalance.** Every pattern exists within a tension. If no opposing concern can be named, the pattern is too absolute.
5. **The working method guides framing, not final output.** It shapes the thinking path. It does not dictate the answer.
6. **User prompt beats the working method** unless it conflicts with steering or safety rules.
7. **Patterns without observed behavioural evidence are not valid method cards.** Speculation is not method.

## Decay

- Patterns lose confidence unless reinforced by new evidence
- Old patterns should be reviewed when project conventions change significantly
